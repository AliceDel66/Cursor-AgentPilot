# 案例：文档任务流程（docs-flow）

> 场景：撰写或更新项目文档（README、手册章节、API 说明）。适用路径：单篇 Cursor 直接写；多文件批量或需要验证命令时交 Codex；资料调研可用 Multitask 并行。骨架，Sprint 4 补全。

## 1. 场景与输入 prompt

文档需求与目标读者。

## 2. Task package

文档任务的验收写法：结构完整性、事实准确性（sources）、可读性标准。

## 3. 派发与执行

写作/更新过程与来源核验记录。

## 4. 验收与 review

按验收清单逐项检查的记录（含 frontmatter 可解析性验证）。

## 5. 复盘

文档类任务与代码类任务在派发上的差异总结。
